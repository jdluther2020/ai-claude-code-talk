The system consists of **four main components**:

## Connections

| From | To | Protocol/Method | Description |
|------|-----|----------------|-------------|
| Client | API Gateway | **HTTP** | The client sends HTTP requests to the API Gateway |

---

## Architecture Summary

- The **API Gateway** serves as the **central node**, connecting to all other components.
- The data flow is **unidirectional** (based on the arrows), originating from the **Client**.
- The API Gateway has **two data retrieval paths**:
  - **Database** — for primary/persistent data storage
  - **Cache (Redis)** — for frequently accessed data, improving performance by reducing direct database queries

## System Architecture — Main Components & Connections

This diagram titled **"System Architecture"** contains **4 main components** connected in a clear data-flow pattern:

### 🧩 Components

| Component | Color | Role |
|-----------|-------|------|
| **Client** | Light Blue | The end-user or front-end application that initiates requests |
| **API Gateway** | Green | The central middleware that routes and processes all incoming requests |
| **Database** | Light Yellow | The persistent data store for the system |
| **Cache** | Red/Coral | A caching layer (Redis) for faster data retrieval |

### 🔗 Connections

1. **Client → API Gateway** (via **HTTP**)
   - The client sends HTTP requests to the API Gateway. This is the system's entry point.

2. **API Gateway → Database** (via **Query**)
   - The API Gateway queries the database directly for data operations (reads/writes).

3. **API Gateway → Cache** (via **Redis**)
   - The API Gateway connects to a Redis-based cache, likely for frequently accessed data to reduce database load and improve response times.

### 📐 Architecture Pattern

This is a classic **cache-aside (or lazy-loading) architecture** where:
- The **API Gateway** serves as the **central hub**, acting as the single point of entry for all client requests.
- It has **two data paths**: one to the **Database** for persistent storage and one to the **Cache (Redis)** for high-speed, temporary data retrieval.
- This design improves **performance** and **scalability** by reducing repeated database queries for frequently requested data.
