## System Architecture Overview

This diagram illustrates a **4-tier system architecture** with the following main components and their connections:

### **Main Components:**

1. **Client** (Light Blue)
   - Entry point for user requests/interactions

2. **API Gateway** (Green)
   - Central middleware/routing component
   - Acts as the intermediary between the client and backend services

3. **Database** (Pale Yellow)
   - Persistent data storage system

4. **Cache** (Red/Coral)
   - In-memory caching layer for improved performance

### **Connections:**

1. **Client → API Gateway** (HTTP)
   - The client communicates with the API Gateway using HTTP protocol
   - Requests flow from the client to the API Gateway

2. **API Gateway → Database** (Query)
   - The API Gateway sends database queries to the Database
   - Retrieves or stores data as needed

3. **API Gateway → Cache** (Redis)
   - The API Gateway interacts with the Cache using Redis
   - Stores frequently accessed data for faster retrieval

### **System Flow:**
- When a client makes an HTTP request, it goes to the API Gateway
- The API Gateway can check the Cache first for quick data retrieval
- If data isn't cached, it queries the Database and updates the Cache
- This architecture optimizes performance by reducing direct database hits and leveraging caching
